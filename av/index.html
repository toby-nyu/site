Audio Examples

Audio Example

This page demonstrates the HTML5 <audio> element with multiple source formats for cross-browser compatibility.

Techniques Used:

  • HTML5 Audio Element: Uses the native <audio> tag with controls attribute
  • Multiple Source Formats: Includes both MP3 and OGG formats to ensure compatibility across different browsers
  • Fallback Content: Provides a message for browsers that don't support the audio element
  • SVG Logo: Implements inline SVG with CSS mask for logo styling
  • Unified Stylesheet: Uses shared av-styles.css with class-based styling for consistency
  • Responsive Navigation: Vertically stacked navigation on mobile with media queries

Custom Audio Player

This page demonstrates a fully custom audio player built with JavaScript, providing complete control over playback without using the default HTML5 audio controls.

Techniques Used:

  • JavaScript Audio API: Controls playback using audio.play(), audio.pause(), and audio.currentTime
  • Custom UI Controls: Styled buttons for play/pause and stop functionality with icon toggling
  • Time Display: Large, prominent time counter showing current position and total duration in MM:SS format
  • Seek Control: Separate range input slider for jumping to any position in the audio track
  • Volume Control: Independent volume slider with live percentage display (0-100%)
  • Event Listeners: timeupdate, loadedmetadata, and ended events for real-time updates
  • Progressive Enhancement: Hidden audio element with JavaScript-controlled interface
  • Custom Styling: Gradient background, circular buttons, and styled range inputs with CSS custom properties
  • Accessibility: ARIA labels and keyboard-friendly controls

Video Examples

Video Example

This page demonstrates a full-screen video background with overlaid text elements, creating an immersive hero section.

Techniques Used:

  • HTML5 Video Element: Autoplay, muted, looping video that fills the viewport
  • Absolute Positioning: Logo, navigation, and heading positioned over video using position: absolute
  • CSS Viewport Units: 100vh and 100dvh for full-height layout
  • Object-fit Cover: Video scales to fill container while maintaining aspect ratio using inset: 0
  • Z-index Layering: Proper stacking of video, logo, navigation, and text overlays
  • Multiple Video Formats: MP4, WebM, and OGV sources for browser compatibility
  • Semi-transparent Overlays: rgba() for readable text and navigation over video
  • Body Margin Control: Uses :has() selector to remove margins on fullscreen video pages
  • Responsive Design: Media queries adjust logo size, font sizes, and navigation layout for mobile

Responsive Video Example

This page demonstrates an advanced implementation with JavaScript-based responsive video source switching based on screen width.

Techniques Used:

  • Dynamic Source Switching: JavaScript function that changes video source based on window.innerWidth
  • Media Optimization: Serves different video resolutions (480p, 720p, 1080p) for different screen sizes
  • Event Listeners: Window resize event with debouncing to prevent excessive function calls
  • Video API: Uses video.load() to reload video when source changes
  • Performance Optimization: Debounce timer delays video reload during resize to improve performance
  • Fixed Navigation: Uses position: fixed for navigation that stays in viewport during scroll
  • Conditional Logic: Breakpoints at 1280px and 1920px for optimal video serving
  • CSS Architecture: Shares unified av-styles.css with ID-based selector (#video-nav) for fixed positioning

HTML5 Multimedia Reference

Audio Element Attributes

Attribute Description Values
controls Displays default playback controls Boolean
autoplay Automatically starts playing when loaded Boolean
loop Repeats audio when finished Boolean
muted Starts with audio muted Boolean
preload Hints to browser how to load audio none | metadata | auto
src URL of the audio file URL

Video Element Attributes

Attribute Description Values
controls Displays default playback controls Boolean
autoplay Automatically starts playing when loaded Boolean
loop Repeats video when finished Boolean
muted Starts with audio muted Boolean
playsinline Plays inline on iOS devices (not fullscreen) Boolean
poster Image displayed before video loads URL
preload Hints to browser how to load video none | metadata | auto
width Width of the video player Pixels or percentage
height Height of the video player Pixels or percentage

JavaScript Audio/Video Properties & Methods

Property/Method Type Description
play() Method Starts playing the media
pause() Method Pauses playback
load() Method Reloads the media element
currentTime Property Get/set playback position (seconds)
duration Property Total length of media (seconds)
volume Property Get/set volume (0.0 to 1.0)
paused Property Boolean indicating if media is paused
ended Property Boolean indicating if playback ended
muted Property Get/set muted state (boolean)
playbackRate Property Get/set playback speed (1.0 = normal)

Common Media Events

Event Description
loadedmetadata Fires when duration and dimensions are available
timeupdate Fires when currentTime changes during playback
play Fires when playback starts
pause Fires when playback is paused
ended Fires when playback reaches the end
volumechange Fires when volume or muted state changes
error Fires when an error occurs loading media

Key Learning Outcomes

Learning Outcomes

  • Understand HTML5 multimedia elements (<audio>, <video>) and their attributes
  • Implement cross-browser compatible media with multiple source formats (MP3, OGG, MP4, WebM)
  • Use CSS positioning (absolute, fixed) and viewport units for immersive layouts
  • Apply JavaScript for responsive media optimization and dynamic source switching
  • Build custom media controls using JavaScript Audio/Video API
  • Create accessible fallback content for older browsers
  • Optimize media delivery based on device capabilities and screen sizes
  • Build unified CSS architecture with class and ID-based variations
  • Implement responsive design with media queries for mobile-first layouts
  • Use modern CSS selectors like :has() for conditional styling
  • Apply negative margins to extend elements beyond parent containers
  • Handle media events (timeupdate, loadedmetadata, ended) for interactive controls
  • Format and display time values in user-friendly formats
  • Create custom range input styling for seek and volume controls
  • Implement progressive enhancement patterns for JavaScript-dependent features
