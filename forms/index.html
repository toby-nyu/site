About This Section

This section demonstrates HTML form creation, styling, validation, and user input handling using both client-side and server-side techniques.

Comprehensive Form Demo

This page showcases a complete sign-up form with multiple input types, organized using fieldsets and enhanced with JavaScript validation and comprehensive user feedback. This example includes extensive comments in the HTML, CSS, and JavaScript files to help you understand how everything works together.

File Structure:

  • HTML: forms/form-demo.html - Fully commented HTML structure
  • CSS: css/forms.css - Organized with section headers and detailed comments
  • JavaScript: js/form-demo.js - Extensively commented validation logic
  • PHP: form2.php (external server) - Processes and displays submitted data

How It Works:

Client-Side (Browser):

  • User fills out the form fields (title, name, email, phone, preferences)
  • JavaScript validates data when user clicks Submit
  • Invalid fields show red borders with error messages
  • Valid fields show green borders
  • Focus states show blue borders for better UX
  • If validation passes, form data is sent to the server

Server-Side (PHP):

  • Form submits via POST to https://tobibrodie.co.uk/nyu/form2.php
  • PHP script receives the form data
  • Data is sanitized using htmlspecialchars() to prevent XSS attacks
  • Script displays a success page showing all submitted data
  • The PHP file handles both this form (8 fields) and the simple form (3 fields)

Key Form Techniques:

  • Form Structure: Uses <form> element with action pointing to external PHP and method="post"
  • Fieldsets and Legends: Groups related form controls using <fieldset> and <legend> for better organization
  • Radio Buttons: Multiple radio inputs with the same name attribute for single selection (title selection)
  • Dynamic Input Creation: JavaScript creates a text input when "Other" title is selected
  • Text Inputs: Standard <input type="text"> for name fields with minimum length validation
  • Email and Tel Inputs: HTML5 input types for better mobile keyboards, validated with regex patterns
  • Select Dropdown: <select> element with <option> elements for preferred contact method
  • Checkbox Input: <input type="checkbox"> for newsletter subscription (optional field)
  • Labels: Proper <label> elements with for attribute linked to input id for accessibility

JavaScript Validation Features:

  • Configuration-Driven: Validation rules defined in a reusable object structure
  • Field Mapping: User-friendly error messages using a field name mapping object
  • Required Fields: Checks all fields except newsletter and otherTitle
  • Minimum Length: Forename and surname must be at least 2 characters
  • Email Pattern: Validates email format (requires @ and 2+ character domain extension)
  • Phone Pattern: Allows digits, spaces, +, -, and () with minimum 7 characters
  • Visual Feedback: CSS classes for focused (.focused), error (.error), and valid (.valid) states
  • Auto-Focus: Automatically focuses and scrolls to the first error field
  • Error Prevention: Removes errors when user focuses on a field to re-enter data

CSS Styling Highlights:

  • Flexbox Layout: Labels and inputs aligned horizontally using flexbox
  • Color-Coded States: Blue for focus (#2196f3), red for errors (#f44336), green for valid (#4caf50)
  • Responsive Design: Stacks vertically on mobile screens (600px breakpoint)
  • SVG Logo Styling: Uses CSS to change SVG fill color and add borders
  • Smooth Transitions: Hover effects on buttons and navigation links

Learning Resources:

To understand this example fully, review the comments in:

  1. form-demo.html - See how form elements are structured and connected
  2. forms.css - Learn about the three-state styling system and responsive design
  3. form-demo.js - Understand the validation flow from user input to error display

Client-Side Validated Form

This page demonstrates a streamlined account creation form with JavaScript validation to ensure data quality before submission.

Techniques Used:

  • Simple Form Structure: Minimal three-field form (name, email, password) for account creation

PHP Server-Side Code

This page displays the complete PHP code used to process form submissions on the external server. This is a learning resource that shows you the server-side code without actually running it.

What You'll Learn:

  • Server-Side Processing: How PHP receives and handles form data submitted via POST
  • Security Best Practices: Using htmlspecialchars() to sanitize user input and prevent XSS attacks
  • Data Validation: Checking if form was submitted and handling missing fields
  • Conditional Rendering: Using PHP to display different HTML based on whether data was submitted
  • Superglobals: Understanding $_SERVER and $_POST arrays
  • Null Coalescing: Using the ?? operator to provide default values

PHP Features Demonstrated:

  • $_SERVER['REQUEST_METHOD'] - Detects form submission method
  • $_POST['fieldname'] - Retrieves form field values
  • htmlspecialchars() - Sanitizes data to prevent malicious code injection
  • isset() - Checks if checkbox or optional fields were submitted
  • ucfirst() - Capitalizes first letter for display formatting
  • PHP conditional statements - Controls what HTML is rendered
  • Inline CSS - All styling contained within the PHP file

Why This Matters:

While client-side JavaScript validation improves user experience, server-side validation with PHP is essential for security. Users can bypass JavaScript validation, so the server must always verify and sanitize data before processing or storing it. This page shows you both the validation logic and the success page that users see after form submission.

  • Apply client-side validation using JavaScript for immediate user feedback
  • Understand the difference between GET and POST methods
  • Style forms with external CSS for consistent presentation
  • Use HTML5 input types for enhanced mobile user experience
  • Prevent form submission with preventDefault() when validation fails
  • Implement radio button groups for mutually exclusive options
  • Create dropdown menus with <select> and <option>
  • Handle form data submission to server-side endpoints
